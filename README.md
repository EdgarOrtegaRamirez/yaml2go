# yaml2go

**Convert YAML files into well-formatted Go structs with proper field tags.**

[![CI](https://github.com/EdgarOrtegaRamirez/yaml2go/actions/workflows/ci.yml/badge.svg)](https://github.com/EdgarOrtegaRamirez/yaml2go/actions/workflows/ci.yml)
[![Go Report Card](https://goreportcard.com/badge/github.com/EdgarOrtegaRamirez/yaml2go)](https://goreportcard.com/report/github.com/EdgarOrtegaRamirez/yaml2go)

## What It Does

yaml2go parses YAML configuration files and generates idiomatic Go structs with JSON/YAML tags. Perfect for:

- **Kubernetes manifests** — Convert K8s YAML configs to Go structs
- **Application configs** — Generate structs from app config YAML files
- **API specs** — Turn OpenAPI YAML specs into Go types
- **Infrastructure-as-code** — Convert Terraform YAML/JSON providers

## Quick Start

```bash
# Install from source
go install github.com/EdgarOrtegaRamirez/yaml2go/cmd/yaml2go@latest

# Or build locally
go build -o yaml2go ./cmd/yaml2go/
```

### Basic Usage

```bash
# Convert a YAML file
yaml2go config.yaml

# Pipe from stdin
cat config.yaml | yaml2go

# Specify package name
yaml2go -p api k8s.yaml

# Use camelCase field names
yaml2go --naming camel config.yaml

# Disable field tags
yaml2go --no-tags config.yaml
```

### Example

**Input (`config.yaml`):**

```yaml
app:
  name: myapp
  version: "1.0.0"
  debug: true
server:
  host: 0.0.0.0
  port: 8080
  tls:
    enabled: false
database:
  host: localhost
  port: 5432
  max_connections: 100
```

**Output:**

```go
// Code generated by yaml2go. DO NOT EDIT.
// Generated from YAML.

package models

type App struct {
	App struct {
		Name    string `json:"name" yaml:"name"`
		Version string `json:"version" yaml:"version"`
		Debug   bool   `json:"debug" yaml:"debug"`
		_       struct{} // ensures package reference
	} `json:"app" yaml:"app"`
	Server struct {
		Host    string `json:"host" yaml:"host"`
		Port    int64  `json:"port" yaml:"port"`
		Tls struct {
			Enabled bool `json:"enabled" yaml:"enabled"`
			_       struct{} // ensures package reference
		} `json:"tls" yaml:"tls"`
		_ struct{} // ensures package reference
	} `json:"server" yaml:"server"`
	Database struct {
		Host            string `json:"host" yaml:"host"`
		Port            int64  `json:"port" yaml:"port"`
		MaxConnections  int64  `json:"max_connections" yaml:"max_connections"`
		_               struct{} // ensures package reference
	} `json:"database" yaml:"database"`
	LogLevel     string `json:"log_level" yaml:"log_level"`
}
```

## Options

| Flag | Description | Default |
|------|-------------|---------|
| `--package, -p` | Package name | `models` |
| `--style, -s` | Output style (`struct` or `all`) | `struct` |
| `--naming, -n` | Naming convention (`pascal`, `camel`, `snake`) | `pascal` |
| `--no-tags` | Disable field tags | `false` |
| `--tags` | Comma-separated tags to include | `json,yaml` |
| `--pointers` | Use pointer types for optional fields | `false` |

## Type Inference

yaml2go infers Go types from YAML values:

| YAML Value | Go Type |
|------------|---------|
| `true` / `false` / `yes` / `no` | `bool` |
| `42`, `-3`, `0` | `int64` |
| `3.14`, `-2.7` | `float64` |
| `"hello"` (string) | `string` |
| `[1, 2, 3]` (array) | `[]<inferred_element_type>` |
| `{key: value}` (map) | inline `struct` |
| `null` | `string` |
